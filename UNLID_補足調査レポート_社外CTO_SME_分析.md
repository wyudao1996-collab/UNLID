- **AI採用アルゴリズム**導入：90秒以内に人材推薦
- **LPS（NZ）を買収**：ニュージーランドの変革支援大手

## 4. UNLIDのSMEターゲット市場——定量的市場調査と選定理由

### 4-1. ターゲット業種の選定フレームワーク

**選定基準（4軸）：**
1. **IT人材不足の深刻度** — 課題が深いほど支払い意欲が高い
2. **AI活用余地** — 非定型・判断業務が多いほどUNLIDの価値が出る
3. **予算の確保しやすさ** — 売上・利益率が高いほどIT投資余地がある
4. **意思決定速度** — 中小企業として意思決定が速い業種を優先

---

### 4-2. ターゲット業種 第1群（最優先：フェーズ1〜2）

#### 【A】IT/情報通信系中小企業（従業員30〜200名）

| 指標 | 数値 | 出典 |
|------|------|------|
| 対象企業数（国内） | 約2万社 | 総務省統計 |
| DX理解度（業界内） | **70.0%**（全業種最高） | 中小企業基盤整備機構2024 |
| IT予算比率 | 売上の2〜5%（全業種最高水準） | JUAS調査 |
| IT人材不足割合（21名以上） | 32.9% | 中小企業基盤整備機構2024 |

**選定理由：**
- DXを理解している = 「AI実装の価値」を説明なしで分かってくれる
- 自社でAIエンジニアを雇えない規模感 = フラクショナルの必要性が明確
- 意思決定が速く、試しやすい予算がある
- UNLIDのクライアントになった場合の紹介・口コミ効果が高い（IT業界のネットワーク密度）

**初期ターゲット企業の特性：**
- 従業員30〜100名のSaaSスタートアップ・受託開発会社・デジタルエージェンシー
- AIを使って生産性を上げたいが「どのAIツールを社内にどう組み込むか」の専門家がいない
- CTOはいるが開発リソースに忙しく、AI実装を横断的に推進できていない

---

#### 【B】専門サービス業（コンサル・会計・法律・採用、従業員30〜150名）

| 指標 | 数値 | 出典 |
|------|------|------|
| 対象企業数（国内推定） | 約1.5万社 | 各業界団体統計 |
| 平均時間単価 | 1〜5万円/時間 | 業界調査 |
| ドキュメント作業比率 | 業務時間の40〜60% | 各社調査 |
| AI代替可能業務比率（文書系） | 60〜80% | McKinsey 2023 |

**選定理由：**
- 文書作成・分析・提案書作成がメイン業務 = AIで最も効率化できる業種
- 時間単価が高い = AI効率化の費用対効果が出やすい
- 「AI実装で1時間節約＝3〜5万円の価値」という計算が成立
- 会計・法律・採用は規制業種 = 人間のチェックが必須 = UNLIDのFDE監督モデルが刺さる

**初期ターゲット企業の特性：**
- 人事コンサル会社、中小税理士法人、採用支援会社、デジタルマーケティング代理店
- 提案書・レポート・分析作業の量が多く、残業が常態化している
- ChatGPTなどを個人で使っている社員がいるが、組織として統一されていない

---

### 4-3. ターゲット業種 第2群（フェーズ2〜3で拡大）

#### 【C】製造業（BtoBメーカー、従業員50〜300名）

| 指標 | 数値 | 出典 |
|------|------|------|
| 対象企業数（国内） | 約10万社（30〜300名規模） | 経産省統計 |
| DX推進率（2024年） | 33.8%（2019年比3倍） | 中小企業庁白書2024 |
| 人手不足による倒産（2024年） | 過去最多 | 帝国データバンク |
| IT予算成長率（2024年） | +5.8%（中堅・中小） | IDC Japan 2024 |

**選定理由：**
- 製造業の最大課題は「人手不足」→ AI・DXによる省力化ニーズが最高潮
- 「2024年問題（労働時間上限規制）」でスケジュール管理・業務効率化が急務
- サプライチェーン管理・在庫管理・品質管理でAI活用余地が大きい
- 保守的な業種のため「信頼できるFDEが現場に入る」モデルが受け入れやすい

**フェーズ2から優先する理由（今すぐ最優先にしない理由）：**
- IT理解度が低い = 教育コストが高い（フェーズ2で事例実績が積まれてから攻める）
- 意思決定が遅い傾向 = フェーズ1の短期PMF検証に不向き
- 「工場・現場」への訪問が必要 = リモート主体のFDEモデルと相性がやや悪い

---

#### 【D】不動産業（仲介・管理会社、従業員30〜200名）

| 指標 | 数値 | 出典 |
|------|------|------|
| SaaS活用進展 | 「入居申込・契約の電子化が進展中」 | 業種別IT投資調査2024 |
| 中小業者のIT投資伸び | 大手に続き拡大中 | 富士キメラ総研 |
| 業務の特性 | 物件提案・契約書作成・顧客対応が多い | 業界調査 |

**選定理由：**
- 物件提案書・契約書・顧客への説明資料など文書業務が多い = AI活用余地大
- 賃貸管理業務の電子化が進んでいる = ITリテラシーが上がっている状態
- 地域密着型の中小業者が多い = フラクショナルモデルの「週1〜2日現場に入る」が刺さる

### 5-3. UNLID適正月額の算出

**導出ロジック：**

```
対象SME：IT予算月25万〜42万円（小規模中小・売上3〜5億円）

UNLID月額が「払いやすい」条件：
  → IT予算の30〜50%以内 = 月8万〜21万円
  → 「試してみよう」と決断できる上限 = 月20〜25万円
  → 正社員ITエンジニア採用と比較（年収450万円 = 月37.5万円）の半額以下

【UNLID最適価格帯】
  エントリー（週1日・AI実装基礎）：月10〜12万円
  スタンダード（週1.5日・複数業務）：月15〜18万円
  プレミアム（週2日・AI戦略立案〜実装）：月20〜25万円
```

**なぜ月15〜20万円がスイートスポットか：**

| 理由 | 説明 |
|------|------|
| **ITとして決裁しやすい** | 月20万円以下 = 多くの中小企業で役員決裁不要の稟議額 |
| **ROIが見えやすい** | 週1.5日のFDEが1業務を自動化できれば、月80〜100時間の節約 = 時給2,000円でも月16〜20万円相当 |
| **競合より明確に安い** | サーキュレーション（月25万円〜）より安い。正社員採用（月30〜40万円超）の半額以下 |
| **継続しやすい** | 月20万円 × 12ヶ月 = 年240万円。IT補助金（IT導入補助金）の対象額内 |

**結論：SME一社あたりの適正月額 = 月15〜20万円（週1〜1.5日稼働）**

- **麻雀プロ**：不完全情報下での期待値判断を専門とする
- **教え子ネットワーク**：若手メンバーの初期調達源として活用予定
- **推定年齢・経験年数**：明記なし（但しレバレジーズ出身の実績人材）

### 批判1：「AI技術力の欠如」——最大の弱点

```
このビジネスの核心は「AI実装のFDE」だ。
しかし羽島の経歴にAI/ML/エンジニアリングの実績は存在しない。

問題：
  ① FDEの品質評価ができるか？
     → CTO経験もエンジニア経験もない創業者が、
       「AI実装を本当にできるFDE」と「できないFDE」を
```

レバレジーズ出身の強みは「B2B営業力」「組織設計理解」「HRの深い知見」だ。しかしこのビジネスが提供するのは**技術的AI実装の成果**であり、営業力は「売れる」ことへの貢献だが「成果を出せる」かどうかとは別の話だ。

```
レバレジーズ出身者の典型的能力：

  ✓ 採用・人材評価
  ✗ AI実装（実務）
  ✗ システム開発・コード設計
  ✗ データ分析・MLエンジニアリング

→ UNLID事業に必要な「AI実装を保証できる責任者」の資格を
  羽島が持てるかは、経歴から読み取れない
```

**致命度：★★★★☆（重大）**

### 批判3：「麻雀プロ」は差別化か、ノイズか

ストーリー価値はあるが、クライアント営業では使いにくい側面がある。

### 批判4：「両面市場の同時立ち上げ」はほぼ不可能レベルの難易度

フラクショナルFDE事業（B2B）と副業若手育成（B2C）を同時立ち上げすることは、理論的には正しいが実務的には二つの別々のスタートアップを同時に創業するのに等しい。

```
B2B側の課題：
  - 最初のクライアント獲得に要する時間：3〜6ヶ月（IT系スタートアップの平均）
  - 実績ゼロの会社へのB2B信頼獲得の困難さ
  - 法人営業には担当者・管理職・役員の3層を突破する必要

B2C側の課題：
  - 副業若手の継続参加にはクエストの量が必要
  - クエストがなければ若手は集まらない（鶏と卵）
  - 「1,000円/時」という報酬は集客上のノイズになる

→ どちらを先に立ち上げるか、または同時に立ち上げるかの
  意思決定が計画から見えない。
  経営資源が分散し、両方が中途半端になるリスク。
```

**致命度：★★★★☆（重大）**

## 7. 公平な視点での蓋然性評価

### 7-1. 羽島がこのビジネスに向いている理由

#### ◎ B2B営業の実証力——最初のクライアントを取れる

レバレジーズ出身者のB2B営業力は業界内で高く評価される。特に：
- **法人顧客への信頼構築**：経営者・人事・営業部門のキーパーソンへのアプローチ経験
- **HRtech・人材領域への深い知見**：UNLIDの人材紹介ビジネス側（スカウト採用）と完全に整合する
- **組織設計の理解**：「中小企業の経営者が何に困っているか」を肌感覚で知っている

**フェーズ1において最も重要なのは「最初の3〜5社を獲得できるか」**だ。この点でレバレジーズ出身の営業力は本物の強みになる。

---

#### ◎ 麻雀プロの「判断プロセスの言語化」能力

完遂ログの「3軸スコア」とFBの型を設計・言語化する能力は、麻雀の「なぜその牌を切ったのか」を言語化するプロセスと同型だ。

```
麻雀プロの強み（ビジネス転用）：
  ① 不確実な情報から「確率的に最善の行動」を選ぶ
  ② その判断を他者（視聴者・生徒）に説明する経験がある
  ③ 「逃げずに最後まで打ち切る」メンタル

→ FDEが副業若手にFBするときの「言語化・型作り」は、
  麻雀解説の経験と能力的に同型。
  これは採用できないユニークな強み。
```

---

#### ◎ 教え子ネットワーク——初期B2Cの鶏と卵問題を部分解決

副業若手（B2C側）を最初に集める問題は、多くの両面市場スタートアップが失敗する原因だ。羽島の場合、既存の教え子ネットワークが存在することで：
- フェーズ0〜1の副業ワーカー6〜10名はリファラルで確保可能
- 彼らが最初のログを積み上げ、採用実績を作れれば「ログの実績」が生まれる

---

#### ◎ このビジネスモデルは「AI技術者でなくても成立する」設計

批判1（AI技術力の欠如）は深刻だが、UNLIDの設計を読み直すと：

```
UNLIDのFDEに求められる能力：
  ✓ AI活用の実務知識（プロンプトエンジニアリング・ツール活用）
  ✓ 業務分析・プロセス設計
  ✓ クライアント企業への説明・合意形成

  ✗ MLモデル開発（不要）
  ✗ サーバー構築・インフラ（不要）
  ✗ データサイエンス（不要）

→ これはMLエンジニアではなく「AI活用コンサルタント」の能力。
  Claude・ChatGPT・Dify等の既存AIツールを組み合わせる能力であり、
  羽島自身がこの水準に到達することは、
  半年〜1年の集中的な実践で可能だ。
```

**つまり、羽島が「AIエンジニア」である必要はない。「AI活用の実践家」になればよい。**

#### ◎ 市場のタイミングが完璧

```
1. AI実装経験のあるCo-founder（または初期CTO）を
```
