①
バックオフィス効率化
Phase 1（導入初月）で即効性を出す最優先カテゴリ。定型業務の削減時間を数値化してROIを証明する。
01
バックオフィス効率化
議事録の自動生成・要点抽出・共有
Notta / Whisper + Claude 工数 3〜5h（導入〜定着） 作業時間 40%削減 難易度 ★☆☆
概要：週1〜2回の会議を録音→文字起こし→AI要約→Slackへ自動投稿するフローを構築。60分の会議後作業が6分に短縮（青森県自治体・不動産会社等での導入事例をベースに設計）。経営会議・週次MTG・クライアント商談など幅広く適用可能。
🔧 導入ステップ
1
録音ツールの選定・試験（Notta / otter.ai / Whisper） Nottaは日本語精度が高く月1,800円〜。無料版で3会議を試し精度を確認する。Whisperはオープンソース版（ローカル実行可）で機密性が高い場合に利用。
判断軸：①日本語精度 ②機密情報の取り扱い ③Slack/Notion連携有無 0.5〜1h
2
参加者への録音告知・同意取得 社内会議は口頭告知でOK。クライアント商談は事前書面または冒頭告知が必須（個人情報保護法への配慮）。「議事録作成のため録音します」の一言で足りる。 0.5h
3
要約プロンプトの設計・テスト（Claudeで実施） 文字起こしテキストをClaudeに貼り付け、下記プロンプトで①決定事項②アクションアイテム③懸案事項を抽出。3〜5回の試行で精度を確認し、プロンプトを微調整する。 1〜2h
4
Slackへの自動投稿設定（Zapier / Notta連携） NottaのSlack連携（有料プラン）またはZapierで「会議終了→要約テキスト生成→Slackポスト」を自動化。初期設定30分。手動でも良い（議事録をコピペしてSlack投稿）。 0.5〜1h
5
テンプレートの固定化・マニュアル化 使用するプロンプトをNotionに保管し「誰でも使えるテンプレート」として共有。担当者が自走できる状態にする。 0.5h
💡 プロンプトテンプレート（Claudeに貼り付けて使用）
# 議事録要約プロンプト
あなたは会議の内容を整理する専門家です。
以下の会議録を読んで、指定のフォーマットで要約してください。

【会議録】
[ここに文字起こしテキストを貼り付ける]

【出力フォーマット】
## ✅ 決定事項（箇条書き・3点以内）
## 📋 アクションアイテム（担当者｜期限｜タスク の表形式）
## ⚠️ 懸案事項・要確認事項（あれば）
## 📅 次回アジェンダ候補（あれば）

# 制約：推測や解釈は書かず、発言内容に忠実に要約すること
# 担当者名が不明な場合は「未定」と記載
⚠️ 注意点・よくある落とし穴
  • 録音ファイルは機密扱い（クラウド保存先の確認必須）
  • Notta無料版は録音時間に月60分制限あり
  • 専門用語・社名は誤認識されやすい→辞書登録が必要
  • 「決定」と「検討中」の混同に注意。プロンプトで明示する
✅ 成功の判断基準（導入後1ヶ月）
  • 議事録作成時間：60分→10分以下
  • 要約精度：参加者が「OK」と言える正確さ
  • Slack投稿：会議終了から1時間以内に共有
  • 担当者が自力でツールを使える
02
バックオフィス効率化
メール返信・提案書の下書き自動生成
Claude / ChatGPT 工数 2〜4h（導入〜定着） 文書作成 60〜70%時短 難易度 ★☆☆
概要：定型メール・営業提案書・お礼状・クレーム対応文など、毎日発生する文書作成を半自動化。「状況説明→下書き生成→人間がレビュー・送信」の流れを定着させる。不動産会社での導入事例では文書作成時間が70%削減。即日導入可能。
🔧 導入ステップ
1
「よく書くメール」の棚卸し（ヒアリング） FDEがオーナー・担当者にヒアリング。「週に何通、どんなメールを書きますか？」月次で繰り返す定型文を10〜15種類リストアップ。優先度：量×時間コストの大きいものから着手。 1h
2
トーン・文体の設定（ペルソナ設計） その会社らしい文体をプロンプトに定義。「丁寧だが堅苦しくない」「短く簡潔に」など実際の過去メールをClaudeに読ませて文体を学習させる。 0.5h
3
メール種別ごとのプロンプトテンプレート作成 「お礼メール」「問い合わせ対応」「見積もり提出」「クレーム対応」各タイプ別に変数（相手名・用件・金額等）を含むテンプレートを作成。Notionに一覧化。 1〜2h
4
実際のメールで10通テスト→品質確認 担当者に実際のメールシチュエーションで試してもらい、修正点を記録。「これはそのまま使える」「ここは毎回直す」を把握してプロンプトを改善。 0.5h
💡 プロンプトテンプレート（メール下書き用）
# 営業メール下書きプロンプト
あなたは私の代わりにビジネスメールを書く優秀なアシスタントです。
以下の情報をもとに、メールの下書きを作成してください。

【メールの目的】：[例：先日の商談のお礼と次回アポ打診]
【相手】：[例：株式会社○○ 営業部 田中様]
【伝えたい内容】：[例：先日ご提示いただいた要件を踏まえ、来週月〜水に打ち合わせをお願いしたい]
【トーン】：丁寧・簡潔・親しみやすい（堅すぎない）

# 制約：
# - 200〜300字程度
# - 過剰な敬語は不要（「ご査収ください」等は使わない）
# - 件名も一緒に提案すること
⚠️ 注意点・よくある落とし穴
  • AIの文章は必ず人間がレビュー・送信（無承認送信は絶対NG）
  • 固有名詞の間違いが出やすい（社名・担当者名を毎回確認）
  • クレーム対応は感情的配慮が必要→AIは下書きまで
  • 社外秘情報をプロンプトに入れない（機密管理）
✅ 成功の判断基準（導入後1ヶ月）
  • 1通あたり作成時間：15分→4分以下
  • AI下書きをそのまま使える割合：60%以上
  • プロンプト集がNotionに10種類以上整備済み
  • 担当者が自分でプロンプトを改変できる

経費精算・請求書OCR処理の自動化
AI-OCR（マネーフォワード）+ Claude 工数 4〜6h（設定〜運用開始） 月20〜30h → 3〜5h（入力ミスほぼゼロ） 難易度 ★★☆
概要：領収書・請求書の手入力を AI-OCR で自動読み取りし、仕訳・承認フローまでを半自動化。マネーフォワードのスマート経費機能（月1,000円〜）が最も導入しやすい。月末の経理作業を月20〜30時間から3〜5時間へ削減。電子帳簿保存法の対応も同時に達成できる。
🔧 導入ステップ
1
現行の経費処理フロー確認（ヒアリング） 現状の「領収書をどう集めてどう入力しているか」を把握。Excelで集計している場合は列構造を確認し、移行の設計をする。月に何枚の領収書・請求書があるか数える。 1h
2
マネーフォワードのスマート経費を試験導入 30日無料トライアル開始。スマホアプリで領収書を撮影→OCR読み取り→自動仕訳提案の流れをテスト。最初の10枚で精度を確認。勘定科目の自動分類精度は約85〜90%。 1〜2h
3
仕訳ルールの設定・よく使う科目の登録 「交通費は旅費交通費」「○○社への支払いは仕入高」等、会社固有のルールをMFに学習させる。設定後は繰り返し請求書を自動分類するようになる。税理士に確認を推奨。 1〜2h
4
Claude との連携：異常値・不備チェックの自動化 月次でMFからCSVエクスポート→Claudeに読み込ませて「金額が異常に高い項目」「未承認のまま残っている項目」「科目が怪しい項目」を自動チェック。下記プロンプトを使用。 1h
5
承認フローのルール化（Slack通知設定） MFのSlack連携で「申請が来たらSlackで承認者に通知」を設定。承認をSlack上で完結させ、月末の確認作業をなくす。 0.5h
💡 プロンプトテンプレート（経費CSV異常値チェック）
# 経費データ異常値チェックプロンプト
以下は今月の経費データCSVです。会計担当者として以下をチェックしてください。

[CSVデータをここに貼り付け]

以下の観点で問題がある行をピックアップしてください：

1. 【高額アラート】1件3万円以上の支出（接待・備品など）
2. 【科目疑義】勘定科目と内容が不一致に見えるもの
3. 【承認なし】承認者欄が空白のもの
4. 【重複疑い】同日・同金額・同店舗の重複申請

# 出力：問題行の一覧表（行番号・金額・問題内容の3列）
# 問題なければ「問題なし」と出力
⚠️ 注意点・よくある落とし穴
  • 手書き領収書はOCR精度が低い→手書きは別途入力
  • 電帳法：スキャン解像度200dpi以上が必要
  • 税理士との連携確認が必須（科目設定・会計ソフトの互換性）
  • MF無料版は機能制限あり→月1,000円の有料プランが実用的
✅ 成功の判断基準（導入後1ヶ月）
  • 月末経理作業時間が半分以下に削減
  • OCR読み取り後の手修正が20%以下
  • 月次の科目分類ミス：導入前比50%以上減少
  • 承認フローがSlack上で完結している
04
バックオフィス効率化
契約書・法務書類の初期AIレビュー
Claude（長文処理に強み） 工数 3〜5h（フロー設計〜定着） 弁護士確認前コスト 70%削減 難易度 ★★☆
概要：毎月発生する業務委託契約書・NDA・賃貸借契約などをClaudeに読ませ、「見落としやすいリスク条項」「不利な条件」「確認すべき箇所」を自動抽出。弁護士への相談前の下調べコストを削減。Claudeは長文処理が特に優れており、30ページの契約書も一括処理可能。
🔧 導入ステップ
1
よく締結する契約書の種類をリストアップ NDA・業務委託・サービス利用規約・賃貸借・雇用契約など。各タイプ別に「特にチェックしたい条項」を整理する（例：NDA→情報範囲・期間・罰則規定）。 0.5h
2
契約書タイプ別チェックリストの設計 各種別で「AIに何をチェックさせるか」のリストを作る。例：業務委託契約なら①瑕疵担保責任②知的財産の帰属③報酬の支払い条件④解除条件⑤競業避止義務。 1〜2h
3
Claudeでの試験レビュー（実際の契約書で3件テスト） 過去に締結した契約書（機密でない部分）でテスト。Claudeの出力と「実際に気になっていた条項」を比較して、プロンプトの精度を調整する。 1〜2h
4
「AI初期チェック→人間確認→弁護士相談」フローの確立 AIのチェック結果を「問題なし・要確認・要弁護士相談」の3段階で分類するルールを決める。「要弁護士相談」以外はAI判断で処理できる体制を構築。 0.5h
💡 プロンプトテンプレート（業務委託契約書レビュー）
# 契約書リスクチェックプロンプト
あなたは法務の知識を持つビジネスアドバイザーです。
以下の業務委託契約書を読み、依頼者（受託側）の立場から
リスクのある条項を指摘してください。

【契約書本文】
[ここに契約書テキストを貼り付け]

【チェック項目】
1. 知的財産権の帰属（受託者に不利な条件はあるか）
2. 瑕疵担保・損害賠償の範囲（過大な責任負担はないか）
3. 報酬の支払い条件と遅延時の対応
4. 解除条件（一方的に不利な解除条項はないか）
5. 競業避止・秘密保持の範囲（過度に広くないか）

# 出力フォーマット：
# 各項目を「問題なし / 要確認 / 要弁護士相談」で評価
# 「要確認」以上は該当条文を引用して問題点を説明
# 最後に総合リスク評価（低・中・高）を出力
⚠️ 注意点・よくある落とし穴
  • AIは法律の専門家ではない。最終判断は必ず人間・弁護士が行う
  • 機密性の高い契約書はClaude.aiではなくAPIで処理（データ学習に使われない設定）
  • 法改正への追随が必要（フリーランス保護新法・電帳法等）
  • AIの出力を「そのまま法的根拠」として使わない
✅ 成功の判断基準（導入後1ヶ月）
  • 契約書の初期確認時間：1件30分→5分以下
  • 弁護士相談件数：重要案件のみに絞り込めている
  • 過去に見落としていたリスク条項を1件以上発見
  • チェックリストが契約種別ごとに整備済み
