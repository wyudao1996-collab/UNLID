## 1. 「3つの市場の交差点」の正確な意味

### 従来の構造の問題点

企業・若者・人材紹介は、現在それぞれ別の市場で別の解決策を使っている。

```
従来の構造：
  企業  →「調査・資料作成・雑務」→ ランサーズ/BPO（安い・品質バラバラ・文脈わからない）
  企業  →「優秀な人材が欲しい」→ 人材紹介（高い・採用前に検証不可）
  若者  →「スキルを積みたい・副収入欲しい」→ ランサーズ（単価低い・成長しない）
```

### UNLIDが「交差点」になると

```
UNLIDが統合する3市場：

  ┌─────────────────────────────────────────────────────┐
  │                                                       │
  │  市場①：BPO・業務委託市場                              │
  │  企業が「摩擦業務の解決」に払う → UNLID の受託収益      │
  │                                                       │
  │  市場②：副業・ギグワーカー市場                          │
  │  若者が「実戦経験＋副収入」を求める → UNLIDが払うコスト   │
  │                                                       │
  │  市場③：人材紹介市場                                   │
  │  企業がギルドメンバーを採用 → UNLID の紹介フィー収益     │
  │                                                       │
  └─────────────────────────────────────────────────────┘

既存プレイヤーはそれぞれ1市場しか見ていない。
UNLIDは3市場を1つのエコシステムで統合する。
```

### なぜ既存プレイヤーが参入しにくいか

| プレイヤー | 参入できない理由 |
|-----------|--------------|
| ランサーズ/クラウドワークス | マーケットプレイスのため品質保証ができない。育成機能なし |
| 一般的なBPO | 定型業務の効率化モデル。摩擦のある非定型業務は不得意 |
| コンサル会社 | シニアが解決するモデルで利益が出る。若手育成しながら解決は利益構造が合わない |
| 人材紹介会社 | 採用前に「実戦で検証する」フェーズを入れると回転率が下がる |

## 2. ToCモデルの再設計——「課金しない、むしろ払う」

### なぜ変えるか：「賃金デッドロック」への直接介入

**データが示す若者の現状（2024〜2025年）：**

- 新卒3年以内離職率：**34.9%**（厚労省、2021年卒）
- 20代男性の転職率：**13.4%**（全世代で最高）
- 20代の副業実施率：**14.4%**（全世代で最低 ← 伸びしろが最も大きい）
- 早期離職を「プラス」と感じる人：**41.3%**（マイナスは29.5%）
- 20代が「働かないといけないと思う年齢」は「働きたい年齢」より**3.3歳上**

**賃金デッドロックの構造：**

```
スキルが積まれない会社に居続ける
  → 市場価値上がらない
    → 賃金上がらない
      → 焦って転職するが履歴書が薄い
        → また同じ環境
```

この人たちに「月1万円払ってください」は刺さらない。
「月20〜30万円もらえて、実績も積める」が正しい答え。

### 新しいToC価値提案（無料どころか収入が得られる）

| 軸 | 内容 |
|------|------|
| **副業収入** | 1クエスト 20〜30万円。ランサーズのタスク作業（数千円）とは桁が違う |
| **完遂ログ** | 「私はこういう摩擦案件をやり遂げた」という第三者検証済みの実績 |
| **師匠のFB** | 自分の判断プロセスを麻雀プロ×レバレジーズ出身の視点で解剖される |
| **次の就職** | クエスト先企業からそのまま引き抜き可能 |

### マズローの全層を同時に刺激する設計

```
安全欲求（どこでも生きられる）：スキルが積まれるから市場価値が上がる
生理的欲求（今すぐお金が必要）：クエスト完遂で20〜30万円の副収入
帰属欲求（仲間が欲しい）：コホート（少人数チーム）での共闘体験
承認欲求（認められたい）：師匠から「お前はここが良かった」とFBされる
自己実現欲求：「不確実な状況で判断して、やり遂げた」という体験
```

## 3. クエスト具体化——何が切り出せるか

### 調査の前提：ランサーズ・クラウドワークスで確認できた市場規模

- ランサーズ：登録企業60万社、依頼数235万件超、350種以上のカテゴリ
- クラウドワークス：登録企業78万社、500万人のワーカー、250種以上の仕事
- 調査・分析カテゴリだけで**12万5千件以上**の案件（ランサーズ）
- リサーチ代行の相場：1件あたり**5万円〜**
- 企業の副業人材受け入れ率：**53.7%**（2024年）

**需要は確実に存在する。UNLIDの差別化は「品質保証と判断力の付加」。**

---

### Tier 1：切り出しやすい・即始められる（Day 1〜6ヶ月）

#### クエスト① 競合・市場の「意思決定に使える」分析レポート

```
企業の依頼例：
「競合A社が急に値下げしてきた。なぜか、どう対応すべきか調べてくれ」
「新規事業を検討しているが、この市場に参入すべきか知りたい」

通常のランサーズ：
  競合情報を収集してリスト化（作業の完了が目的）

UNLIDのクエスト：
  「なぜ値下げしたか」の仮説を複数立て → 証拠を収集 → 「経営判断に使える示唆」を整理
  ＋師匠のFB：「この仮説を立てた根拠は？」「この数字は信頼できるソースか？」

3つの誠実プロセス評価：
  情報の網羅性：どこまで調べるかの判断
  相手への配慮：読む人（経営者）が何を知りたいかを考えた構成
  目的への執着：「もう少し調べればもっと分かる」を諦めなかったか

単価目安：50〜150万円（ランサーズ比3〜10倍、でも意思決定に使えるなら割安）
```

#### クエスト② 新規取引先・パートナー候補の信用調査＋提案設計

```
企業の依頼例：
「この会社と提携したい。取引して大丈夫か、相手が何を求めているか分からない」

UNLIDのクエスト：
  公開情報・登記・SNS・ニュース等から信用情報を整理
  → 「相手は何を欲しがっているか」を推論
  → 提案の切り口・構成案の作成

摩擦ポイント：
  不完全情報の中での判断。「見えていないリスクをどう扱うか」
  ＝麻雀プロの思考（不完全情報下の期待値判断）がFBに活きる

単価目安：30〜100万円
```

#### クエスト③ 社内稟議・投資家説明用の「通る」資料作成

```
企業の依頼例：
「事業部長が作った稟議書が役員会で通らない。どこが問題で、どう直すべきか」

UNLIDのクエスト：
  資料の論理構造の問題点を整理
  → 「この稟議を承認する側（役員）の疑問・懸念」を代弁
  → 改訂案の作成＋変更箇所の説明

摩擦ポイント：
  「誰が読むか」「何を不安に思うか」の想像力が必要
  答えは1つではない（CFOとCOOでは気にするポイントが違う）

単価目安：30〜80万円（緊急性高い場合は高単価）
```

### Tier 2：少し深い・2〜3ヶ月のクエスト（6ヶ月〜2年）

#### クエスト④ 採用ブランディング「なぜここで働くべきか」の言語化

```
企業の依頼例：
「求人を出しても応募が来ない。うちの何が魅力なのか、どう伝えればいいか分からない」

UNLIDのクエスト：
  社内インタビュー（3〜5名）→ 強みの言語化
  → 採用ページのコピー・構成案
  → 競合求人との差別化ポイント整理

摩擦ポイント：
  社員が「当たり前」と思っていることの中に価値がある
  引き出すには心理的安全と丁寧なヒアリングが必要
  ＝表層の言葉ではなく「なぜそこで働き続けているか」の本音を掘る

単価目安：50〜150万円
```

#### クエスト⑤ 顧客・ユーザーインタビューの設計・実施・示唆抽出

```
企業の依頼例：
「顧客が何に困っているか、もっと深く知りたい。でも社員が聞くと本音が出ない」

UNLIDのクエスト：
  インタビュー設計 → 実施（社員でない第三者だから本音が出やすい）
  → 「表面の言葉の裏にある本音」の整理 → 示唆レポート

摩擦ポイント：
  「聞いたことをまとめる」ではなく「なぜそう言ったか」を掘り下げる力
  ＝不完全情報から本質を読む力（麻雀的思考が活きる）

単価目安：80〜200万円
```

#### クエスト⑥ トラブル・クレーム後の「誠実さが伝わる」対応設計

```
企業の依頼例：
「取引先に迷惑をかけた。どう謝るか、再発防止をどう説明するかを一緒に考えてほしい」

UNLIDのクエスト：
  何が問題だったかの整理
  → 「相手は何に怒っているか」の分析（事実への怒りか、態度への怒りか）
  → 対応の言語化・コミュニケーション設計

摩擦ポイント：
  感情を含む状況での判断力。正論だけでは解決しない。
  「謝り方が下手」がさらに関係を悪化させるリスクを回避する

単価目安：50〜120万円（緊急性高いため単価が取りやすい）
```

### Tier 3：将来の本丸（信頼が積まれてから）

#### クエスト⑦ 部署間・ステークホルダー間の合意形成のための地ならし

```
企業の依頼例：
「A部署とB部署が対立していてプロジェクトが止まっている。調停の下準備をしてほしい」

UNLIDのクエスト：
  双方の主張・本音の整理（個別ヒアリング）
  → 合意しやすい落とし所の設計
  → 議論の場の構造・進め方の設計

摩擦ポイント：
  最も高い「逃げない執着」が必要
  政治的文脈への深い理解・中立性の維持も必要

単価目安：100〜300万円
注意：Day 1から取るのは難しい。Tier 1〜2の実績が積まれれば自然に流れてくる
```

### なぜランサーズ・クラウドワークスで解決しないのか

| 比較軸 | ランサーズ/CW | UNLID |
|--------|-------------|-------|
| 発注者が求めるもの | 「この作業をやってください」（作業の明確化が前提） | 「この問題、一緒に考えて解決してください」 |
| ワーカーの動き方 | 指示通りに動く | 「何を調べるべきか」から考える |
| 品質保証 | なし（クライアントが選ぶ） | 師匠のFBが入る構造的品質保証 |
| アウトプット | 成果物の納品 | 成果物＋判断プロセスの可視化 |
| 若者の得るもの | 報酬のみ | 報酬＋完遂ログ＋師匠のFB |
| 企業が得るもの | タスクの完了 | 課題解決＋採用前候補の実戦検証 |

## 4. 市場ニーズ検証——本当に売れるか

### ToBのニーズ

**答え：ある。ただし「摩擦業務」という言葉では売れない。**

**データの裏付け：**
- ランサーズ登録企業60万社、クラウドワークス78万社 → 業務外注需要は巨大
- 調査・分析カテゴリ単体で12万件超の案件 → この類の仕事は確実に発注される
- OJT崩壊・育成放置の実態 → 「育てたいが時間がない」は構造的問題
- 企業の副業人材受け入れ率53.7% → 外部人材を使う素地がある

**売れる言葉の設計：**

```
×「摩擦のある業務を受託します」（内部用語）
○「社内に『誰かがやらないといけないけど、手が回っていない調査・資料・対応』は
   ありませんか？うちの若手が師匠のもとで、半分コンサル・半分実務で入ります。
   そのまま気に入れば採用もできます。」
```

**単価の根拠：**

| 比較対象 | 単価 | UNLIDの単価 |
|---------|------|-----------|
| ランサーズのリサーチ代行 | 5〜20万円 | 50〜150万円 |
| 零細コンサル | 50〜100万円 | 50〜200万円 |
| 新卒1名の採用コスト | 50〜100万円 | 紹介フィー：年収×35% |

**正直な懸念：**

初期の最大ボトルネックは「クエストの切り出し」。企業は「この業務を外に出していいか」の判断基準がない。羽島氏自身がコンサルとして入り、「棚卸し」から始めることが必須。最初の3〜5社は人脈からの直接営業で実績を作ってから横展開する。

---

### ToCのニーズ

**答え：ある。「払う」モデルの方が圧倒的に刺さる。**

**データの裏付け：**
- 20代の副業実施率14.4%（最低）→ まだ市場が開拓されていない。伸びしろ大
- 副業市場全体：フリーランス人口1,303万人、経済規模20兆3,200億円（2024年）
- 正社員副業市場：約0.8兆円（まだ立ち上がり段階）
- 企業の副業人材受け入れ53.7% → 若者が副業できる環境は整いつつある

**「賃金デッドロック」を断ち切る設計：**

```
現状の若者：
  ランサーズで文字起こし → 1,000円/本 → スキルにならない
  eラーニングを買う → 積ん読 → 成長実感ゼロ
  転職活動 → 「実績が薄い」と言われる → また同じ職場

UNLIDの場合：
  クエストを完遂 → 20〜30万円の副収入（生理的欲求を満たす）
  + 「このフェーズで迷ったときに〇〇と判断し、FBで△△と指摘された」完遂ログ
  → 次の転職活動で「何をやってきたか」が具体的に語れる（市場価値が上がる）
```

**若者の文脈（Gemini対話から得た重要な洞察）：**

```
若年層のマズロー的変化：
  （初期）自己実現欲求：「やりたいことをやりたい」で転職
    ↓
  （5年後）安全欲求への回帰：「このままだと食えなくなる」という切迫感
    ↓
  （結果）「どこでも生きていけるスキル」を求めて流動化を繰り返す
    ↓
  しかし「スキルを身につけられる場所がない」→ デッドロック継続

UNLIDが提供するもの：
  スキルの向上（どこでも生きられる安全欲求）
  + 副収入（今すぐの生理的欲求）
  + 仲間との共闘（帰属・承認欲求）
  = 3層の欲求を同時に満たす唯一の場
```

## 5. 修正後の収益モデル

### フロー図

```
【企業】
  ↓ 受託費 100〜300万円/クエスト
【UNLID】
  ↓ 若手への副業報酬          ↓ 羽島氏FB料＋マージン
【若手ギルドメンバー】      【UNLID運営】
  ↓（採用成立時）
【人材紹介フィー 年収×35%】→ UNLID
```

### 収益源の整理

| 収益源 | モデル | 単価感 | 初期〜3年 |
|--------|--------|--------|---------|
| B2B受託費（メイン） | クエスト単位 | 50〜300万円/件 | 10〜25億円SOM |
| 人材紹介フィー | 採用成立時成功報酬 | 年収400万×35%=140万円/件 | 0.5〜1.5億円 |
| B2B顧問契約 | 優先アクセス・月額 | 30〜100万円/月 | 将来の安定収益 |

**ToCからの直接収益はゼロ（むしろ支払う）**

若手への副業報酬（20〜30万円/クエスト）は、B2B受託費の一部として企業が間接的に負担する。若者は「無償のリソース」ではなく「副業収入を得るプロフェッショナル」として設計する。

## 6. Day 1で検証すべきこと

優先度順に3つ：

1. **クエストの切り出し検証**
   羽島氏の人脈から企業1〜3社にヒアリング。「社内に放置されている調査・資料・対応業務」を棚卸しする。これ自体がコンサルティング価値になる。

2. **若手の反応検証**
   現在の「教え子」候補（キャリア支援中の若手）に「クエストをやってもらう、20〜30万円を払う」と伝えた時の反応。「本当に？」という反応か、「何をすればいいか」という反応か。

3. **完遂ログのフォーマット設計**
   1クエストを走らせ、「どう記録するか」を実験する。3軸（情報の網羅性・配慮・執着）でのFBを実際に行い、言語化できるかを確認する。
