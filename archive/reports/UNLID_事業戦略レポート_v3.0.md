## エグゼクティブサマリー

UNLIDが狙うのは、**日本の中小企業（SME）のAI実装を「現場に入って成果を出す」フラクショナルFDEで支援し、その監督下で副業若手ワーカーが実務を担って実績ログを積み、最終的にスカウト型人材紹介で収益化するエコシステム**だ。将来はその実績ログをGitHubのビジネス職版SaaSとして外販する。

| 論点 | 結論 |
|------|------|
| **何をやるか** | フラクショナルFDE（週1〜2日複数社掛け持ち）がSMEのAI実装支援。副業若手が実務を担当 |
| **市場の空白** | LayerX（エンタープライズ特化）・サーキュレーション（既存プロ人材）・顧問名鑑（OB中心）のどれもSME×AI×若手育成×完遂ログを組み合わせたモデルは存在しない |
| **TAM** | 5,000億〜9,000億円（AI業務支援+人材紹介市場） |
| **SAM** | 280億〜560億円 |
| **SOM（3〜5年）** | 4〜6億円（FDE数がボトルネック）／認定FDE拡張で10〜15億円 |
| **将来SaaS** | 完遂ログのGitHub化→採用企業が閲覧課金する採用プラットフォームへ |

## 1. 事業の本質と独自性

### 1-1. 一言で言えば

> 「中小企業のAI実装を、現場に入る正社員FDEが責任を持って成果を出す。その監督下で副業若手が実務を担い、実績ログを積む。ログが証明書になり、採用に繋がる」

### 1-2. 3市場の交差点

```
市場①：BPO・業務委託（TAM 4〜5兆円）
  企業が「AI実装・業務改善」の解決に払う金

市場②：副業・ギグワーカー（TAM 20兆円）
  若者が「実戦経験＋副収入」を求める流れ

市場③：人材紹介（TAM 3,000〜4,000億円）
  企業が即戦力を採用するために払うフィー

→ 3市場を一つのエコシステムで統合する
  既存プレイヤーはそれぞれ1市場しか見ていない
```

### 1-3. Palantir FDEモデルとの違い

Palantirは自社プラットフォーム（Foundry/AIP）を持ち込み、大企業・政府に年数億〜数十億円で常駐する。UNLIDは：

| 比較軸 | Palantir | UNLID |
|--------|---------|-------|
| 対象 | 大企業・政府機関 | 中小企業（30〜300名） |
| 単価 | 年数億〜数十億円 | 月10〜25万円（週1〜2日） |
| ツール | 自社Foundry/AIP必須 | 既存AI（Claude/GPT/Dify等）を組み合わせ |
| 副業ワーカー | なし | FDE監督下で若手が参加 |
| 人材紹介連携 | なし | 完遂ログ→スカウト採用へ |

## 2. なぜ今か——構造的追い風

### 2-1. 中小企業のAI実装ニーズが急拡大（データ）

| 指標 | 数値 | 出典 |
|------|------|------|
| 国内生成AI市場（2024年） | **1,016億円**（初の1,000億円超え） | IDC Japan |
| 国内生成AI市場（2028年予測） | **8,028億円** | IDC Japan |
| 中小企業のAI方針未策定 | **約50%** | 総務省 令和7年版情報通信白書 |
| 中小企業（300名未満）の全社AI導入率 | **わずか1.3%** | 各社調査統計 |
| 大企業との導入格差 | **15倍**（5,000名超の企業は19.0%） | 同上 |

```
中小企業の最大の導入障壁（総務省調査）：
  1位：「効果的な活用方法がわからない」
  2位：セキュリティリスク
  3位：コスト（初期・ランニング）

→ 「方法を教えてくれる人が現場にいない」
  ＝ UNLIDのフラクショナルFDEが埋める穴
```

### 2-2. 外部CTO・フラクショナル市場の確立

```
外部CTO・技術顧問の単価相場（2024〜2025年）：
  副業CTO（週1〜2回稼働）：月30〜50万円
  社外CTO顧問サービス：月10万〜120万円（最多：月60万円）
  正社員CTO採用コスト：年2,000万円超（社保込）

→ 「月60万円の外部CTOは普通に売れている」
  UNLIDはこれより安い「月10〜25万円」で
  AI実装に特化したFDEを提供する
```

### 2-3. 若年層の「賃金デッドロック」と副業需要

| 指標 | 数値 | 出典 |
|------|------|------|
| 新卒3年以内離職率（大卒） | **34.9%** | 厚生労働省（2021年卒） |
| 20代副業実施率 | **14.4%（全世代最低）** | パーソルキャリア2024 |
| 副業市場規模 | **20兆円** | ランサーズ「フリーランス実態調査2024」 |
| フリーランス人口 | **1,303万人** | 同上 |

**20代の副業実施率が全世代最低＝開拓余地が最大。** 既存副業（ランサーズ等）は単価低・スキルにならない・実績が証明できないという課題がある。

### 2-4. OJT崩壊——育てられない構造的現実

| 指標 | 数値 | 出典 |
|------|------|------|
| 計画的OJTを受けた新入社員 | **40.9%→30.3%に低下** | パーソル総合研究所「OJTに関する定量調査」2025年1月 |
| Off-JTを受けられなかった若手 | **約46.5%** | 同上 |
| OJT担当者が「業務過多」 | **37.6%の企業が課題** | 同上 |

```
OJT崩壊の構造的理由：
  ① ハラスメント意識で指導を萎縮
  ② 業務過多で育成時間が消失
  ③ OJT担当が入社2〜3年目の若手（ベテラン不在）
  ④ 「コストとして見えない」ため削減されやすい

→ 会社に育ててもらえない若者が、
  外部で実戦経験を積む場を求める構造が加速
```

## 4. 競合分析——真の競合は誰か

### 4-1. 日本の主要競合

| 企業 | ビジネスモデル | 単価 | UNLIDとの決定的な違い |
|------|-------------|------|-------------------|
| **LayerX Ai Workforce** | 自社AIエージェント基盤（Ai Workforce）を持ち込み、FDEがエンタープライズに常駐実装 | 非公開（エンタープライズ契約） | 中小企業非対応。副業若手なし。自社ツール必須 |
| **サーキュレーション** | 既存プロ人材（15,000名超）を月額25万円〜で企業に派遣。年商88億円 | 月額25万円〜（最低6ヶ月） | 「既に完成したプロ人材」が前提。若手育成・完遂ログなし |
| **顧問名鑑** | 上場企業OB役員・部長（24,000名）を顧問として紹介 | 年間契約（要問い合わせ） | OB中心。AI実装特化ではない。若手育成なし |
| **HiPro（パーソル）** | プロ人材のフリーランス紹介・副業マッチング | 月25万円〜 | マーケットプレイス型。品質保証・育成なし |
| **技術顧問系各社** | AI/IT分野の外部CTO・技術顧問（個人） | 月10〜120万円 | 個人依存。チーム体制なし。副業育成なし |

### 4-2. グローバルの類似モデル

| 企業 | 内容 | 日本展開 |
|------|------|---------|
| **Toptal** | 世界のトップ3%のフリーランサーをマッチング | 未進出 |
| **Expert360（豪）** | プロ人材のフラクショナル活用支援 | 未進出 |

| **Palantir FDE** | 大企業・政府向けAI実装常駐 | 日本大企業のみ（富士通等） |

**結論：「SME × AI実装 × フラクショナル × 副業若手育成 × 完遂ログ × スカウト人材紹介」を組み合わせたモデルは、日本に存在しない。**

## 5. 参入障壁——なぜ真似されにくいか

### 5-1. 構造的参入障壁

| 障壁 | 内容 |
|------|------|
| **FDE品質の育成コスト** | 「現場で責任を持てるFDE」を育てるには最低1〜2年。スキルセットが再現困難 |
| **完遂ログの蓄積** | 1,000件以上のログがないとSaaS化の価値が出ない。先行者が圧倒的有利 |
| **両面ネットワーク効果** | B2B（企業）とB2C（若手ワーカー）の両方を同時に持つ必要がある。どちらかだけでは機能しない |
| **創業者の信頼資産** | レバレジーズ出身×麻雀プロというユニークな組み合わせ。「判断プロセスのFB」に不可欠 |
| **法的設計の複雑さ** | 偽装請負・有料職業紹介・業務委託の3法を同時にクリアする設計が模倣コスト高 |

### 5-2. なぜ既存プレイヤーが参入しにくいか

| プレイヤー | 参入できない構造的理由 |
|-----------|-------------------|
| **サーキュレーション** | 「既成プロ人材」が商品なので「若手育成」を入れると単価が下がり収益モデルが崩れる |
| **LayerX** | 自社ツール（Ai Workforce）がないとサービスが成立しない。SMEは対象外 |
| **顧問名鑑** | OBのプライドと「若手と同格」のポジションが矛盾。モデル転換不可 |
| **研修会社** | アウトプット（成果）での課金は事業部コストが膨大。インプット（講義）に戻る構造 |
| **クラウドソーシング** | 品質保証と育成はマーケットプレイス型と構造上矛盾する |

## 6. サービス設計（フラクショナルFDE×副業ワーカー）

### 6-1. フラクショナルFDEの稼働設計

```
FDE正社員1名が4〜5社を掛け持ち
  各社：週1.5日（月6日 ≒ 48時間）の関与

【企業側コスト】
  月10〜25万円
  ※正社員FDE採用（年収600〜900万円+社保≒年2,000万円超）の
    1/5〜1/10で「専任に近い」支援が受けられる

【UNLID側収益（FDE1名から）】
  4社 × 月15万円 = 月60万円
  - FDE人件費（月35〜45万円）
  = 粗利20万円/名

【商流の設計（偽装請負対策）】
  クライアント企業 → UNLID（業務委託契約、元請け）
                       ↓
                FDE正社員（管理責任者として現場に在籍）
                       ↓
               副業ワーカー（UNLIDから業務委託）

  ✓ クライアントから副業ワーカーへの直接指示を禁止
  ✓ 指揮命令はUNLID→ワーカー（FDE正社員経由）
```

### 6-2. 副業ワーカーが担うクエスト（現実的な範囲）

**できること（FDE監督下）：**

| クエスト | 難易度 | 単価目安 |
|---------|--------|---------|
| AI出力の「人間仕上げ」（資料・文章・提案） | 即対応可 | 月3〜15万円（月額） |
| 新入社員の営業ロールプレイ相手+FB | 即対応可 | 半日2〜5万円 |
| 顧客・ユーザーインタビュー実施・整理 | 3〜6ヶ月 | 案件30〜100万円 |
| 採用ブランディング調査・1次取材 | 3〜6ヶ月 | 案件50〜150万円 |
| 社内研修・ワークショップのファシリテーション | 3〜6ヶ月 | 2〜30万円/回 |

**できないこと（削除済み）：**
- ✗ 稟議・投資家説明用資料（文脈理解が必要）
- ✗ 部署間・ステークホルダー間の合意調整（SES/コンサルPO領域）

### 6-3. 完遂ログの設計

```
【1クエスト完了時に記録する5項目】

1. クエスト種別・業種・規模感（守秘義務Tier1）
   例：「AI出力仕上げ / IT系 / 従業員50〜100名」

2. 自分の担当役割と時間
   例：「まとめ役 / 約20時間」

3. 詰まったポイントと突破法（本人記述）
   例：「インタビュー相手が本音を話してくれなかった。
        FDEから"相手に断る理由はない"と指摘を受け
        別角度で聞き直したら話してくれた」

4. FDEによる3軸スコア
   - 情報の網羅性：★★★★☆（4/5）
   - 相手への配慮：★★★☆☆（3/5）
   - 目的への執着：★★★★★（5/5）
   + 一言コメント

5. 次回改善点（本人記述）

→ これが積み上がると「GitHubの草（Active Contributor）」の
  ビジネス職版になる
```

**守秘義務Tier制：**

| Tier | 公開内容 | 条件 |
|------|---------|------|
| Tier 1 | 業種・規模感・課題カテゴリのみ | 全クエストで標準適用 |
| Tier 2 | 企業名（匿名化）＋成果の概要 | クライアント承認後 |
| Tier 3 | 企業名公開＋詳細ケーススタディ | クライアントの採用ブランディング目的 |

## 7. 収益モデル

### 7-1. 全体の資金フロー

```
【企業（クライアント）】
  ↓ 業務委託費（月10〜25万円）
【UNLID】
  ├─ FDE正社員への給与（月35〜45万円）
  ├─ 副業ワーカーへの報酬（1,000〜3,000円/時）
  └─ UNLID運営マージン（受託費の20〜30%）

【採用成立時】
  企業がワーカーをスカウト採用
  → 人材紹介フィー（年収×35%＝約140万円/件）→ UNLID
```

### 7-2. 3つの収益源

| 収益源 | モデル | 単価 | フェーズ3の規模感 |
|--------|--------|------|----------------|

| **人材紹介フィー** | 採用成立時成功報酬 | 年収400万×35%=**140万円/件** | 年50件=7,000万円 |
| **プロジェクト型** | スポット | 80〜200万円/件 | 年30件=2,400万〜6,000万円 |

## 8. 将来構想——完遂ログのSaaS化（GitHubのビジネス職版）

### 8-1. GitHubの収益モデルから学ぶ

```
GitHubの収益構造：
  「個人に無料で使わせて習慣化させ、
   その人が企業に入ったときに企業から回収する」

  具体的な収益源：
  ① GitHub Enterprise：月21ドル/ユーザー（メイン収益）
  ② GitHub Copilot：月19ドル/ユーザー（急成長中）
  ③ GitHub Actions：従量課金
  → Microsoftが2018年に約7,500億円で買収
```

### 8-2. 日本にビジネス職版GitHubは存在しない

| サービス | 何ができるか | 欠点 |
|---------|-----------|------|
| Wantedly | 経験・プロジェクトを書いて採用に繋げる | 全部**自己申告**。第三者検証なし |
| LinkedIn | スキル・推薦コメント | Endorsementは形骸化 |
| YOUTRUST | 信頼ネットワーク×副業転職 | 実績の「中身」は見えない |
| Findy/Lapras | GitHub活動を集約して採用に活用 | **エンジニア専用**。ビジネス職は対象外 |

**→「FDEが3軸スコアをつけた第三者検証付きの実績記録」はどこにも存在しない。**

### 8-3. SaaS化の収益モデル（フェーズ4以降）

```
前提条件：完遂ログ累計1,000件超

① 採用企業からの閲覧課金（GitHub Enterprise型）
   「完遂ログDBを検索できる採用担当者アカウント」
   → 月3〜10万円/社

② スカウト成立時の成功報酬（フェーズ1から機能）
   完遂ログ経由で採用成立 → 紹介フィー（年収×35%）

③ 企業向けSaaS（自社若手管理）
   「自社の若手社員の完遂ログを社内で管理したい」
   → 月5,000〜1万円/ユーザー
   人事部門の研修効果可視化ニーズに対応
```

## 9. リスクと対策

### 9-1. 法的リスク（最重要）

| リスク | 深刻度 | 内容 | 対策 |
|--------|--------|------|------|
| **偽装請負** | ★★★（最重要） | FDE+副業ワーカーが客先で働く場合、クライアントが直接指揮命令を出すと違法 | FDE正社員が「管理責任者」として現場に在籍。クライアント→副業ワーカーへの直接指示を禁止。契約書に明記 |
| **有料職業紹介業の未取得** | ★★★（最重要） | 人材紹介フィーは職業安定法上の許可が必要 | フェーズ0〜1で申請。許可取得前はフィーを収益に計上しない |
| **守秘義務・情報漏洩** | ★★★ | 完遂ログに業務情報を記録するリスク | 守秘義務Tier制。NDA標準化。情報アクセス権限の設計 |
| **競業避止義務** | ★★ | FDEが複数社掛け持ち時の情報持ち出し | 同業種2社以上の同時担当禁止。情報アクセス権限の分離 |
| **副業ワーカーの雇用判定** | ★★ | 実態が雇用と判定されると労働法が適用 | 指揮命令の明確な排除。成果物で評価。稼働時間の管理はしない |

### 9-2. ビジネスリスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| **FDE採用・育成の難しさ** | ★★★ | 創業者自身が最初のFDEとして動く。採用はリファラル中心 |
| **中小企業の認知・信頼獲得** | ★★★ | 最初の3〜5社は創業者の人脈から。成果をケーススタディ化 |
| **完遂ログの蓄積スピード** | ★★ | フェーズ1から記録を開始。500件到達をSaaS化検討の条件とする |
| **副業ワーカーの品質バラつき** | ★★ | FDE監督下で品質担保。失敗も完遂ログに記録して学習資産化 |
| **AIの進化で仕事がなくなる** | ★★ | クエスト定義を「AI代替困難」＋「人間の信頼関係が必要」軸で定期見直し |
| **スケールのボトルネック** | ★★ | FDE正社員数が上限。認定FDE（外部）制度でスケール解決 |

## 10. ロードマップ

```
フェーズ0（〜3ヶ月）：法的精査と事業設計
  ✓ 弁護士・社労士による偽装請負リスク精査（最優先）
  ✓ 有料職業紹介業の申請スケジュール確認
  ✓ クエスト類型の確定（できること・できないことを整理）
  ✓ 完遂ログのフォーマット設計
  ✓ 初期クライアント3〜5社のヒアリング（創業者の人脈から）
  ✓ 初期副業ワーカー6〜10名の選定（既存の教え子から）

フェーズ1（3〜12ヶ月）：PMF検証
  目標：B2B 8〜12社・年収〜2,200万円・ワーカー20〜50名
  重点：創業者自身がFDEとして動く。完遂ログの型を固める

フェーズ2（1〜3年）：スケール証明
  目標：B2B 40〜60社・年収1.0〜1.7億円・ワーカー200〜500名
  重点：FDE 2〜5名採用。有料職業紹介業の本格運用

フェーズ3（3〜5年）：エコシステム確立
  目標：B2B 120〜200社・年収4〜6億円・ワーカー数千名
  重点：認定外部FDE制度の開始。完遂ログ1,000件突破
  → SaaS化の可否判断

フェーズ4（5年〜）：SaaS化
  完遂ログプラットフォームを採用企業に開放
  → GitHub型の採用プラットフォームとして確立
  → 目標売上：15〜30億円圏
```
