## 1. 事業の本質

### 1-1. 一言で言えば

> 企業が抱える「誰かがやらないといけないが、手が回っていない仕事」を、若手プロフェッショナルが3〜4人のチームで完遂する。その過程で生まれる「逃げなかった実績」が、AI時代に最も価値のある信頼の証明書になる。

### 1-2. 既存の3市場が解決できていないこと

```
【現在の構造と限界】

  企業 →「調査・資料・対応」→ ランサーズ/BPO
            作業は代替できる。文脈・判断・感情は代替できない

  企業 →「優秀な人材が欲しい」→ 人材紹介
            採用「前」に一緒に働く仕組みがない

  若者 →「スキル×副収入」→ ランサーズ
            単価低い・スキルにならない・実績が証明できない

【UNLIDが3市場の交差点になると】

  企業 → UNLIDに「業務委託費」として月3〜20万円を支払う
       → 副産物として「優秀な若手と実戦で相性確認」ができる
  若者 → UNLIDで「副業報酬（1,000〜5,000円/時）」を得ながら
       → 「AI時代に価値ある実績ログ」を積む
  採用成立時 → UNLIDが人材紹介フィー（年収×35%）を受け取る

この3つを同時に機能させる構造を、既存の3市場いずれも持っていない。
```

### 1-3. 研修会社との決定的な違い

| 比較軸 | 研修会社 | UNLID |
|--------|---------|-------|
| 提供するもの | コンテンツ・講義・eラーニング | 生の実務（クエスト）＋師匠FB |
| 育成の方法 | Input（知識を入れる） | Output（実務を完遂させる） |
| 評価の対象 | 受講完了・テスト点数 | 完遂ログ：網羅性・配慮・執着 |
| 若手にとっての意味 | 「やらされた研修」 | 「自分の市場価値を証明する実績」 |
| 企業にとっての意味 | 社員スキルアップ（抽象的） | 現場課題の解消（具体的）＋採用前検証 |
| AI時代の耐性 | AIが代替可能なスキルを教えている | **AIが代替できない力を鍛えている** |
| 予算の名目 | 研修費（人事部予算） | **業務委託費（事業部予算）** |

**「業務委託費」で入ることが重要だ。** 研修費は年度予算内での稟議が必要で動きが鈍い。業務委託費は事業部が案件単位で判断でき、意思決定が速く、成果が見えやすい。

## 2. なぜ今か

### 2-1. AIによる「知識の無価値化」

2022年のChatGPT登場以降、「知識を持っていること」の市場価値は急速に下落した。

```
2020年以前：知識 × 経験 = 市場価値 → 研修に金を払う意味があった
2026年現在：知識はAIが無料提供 → 知識インプットへの投資価値が薄れた
          → 「判断し、完遂し、相手の感情を配慮できる人間」の価値が逆に急騰
```

**AI代替可能性の境界線**

| タスク | AI代替可能性 |
|--------|------------|
| 情報の整理・要約 | 高（すでに進行中） |
| 論理的分析・数値処理 | 高 |
| 定型コミュニケーション | 中 |
| **利害調整・合意形成** | **低** |
| **感情を伴う対応** | **低** |
| **不確実な中での判断** | **低** |
| **人前に立つ・場を作る** | **ほぼ不可能** |

UNLIDが鍛えるのはすべて「低・ほぼ不可能」の領域だ。

### 2-2. 日本若年層の「賃金デッドロック」

```
スキルが積まれない会社に居続ける
  → 市場価値上がらない
    → 賃金上がらない
      → 焦って転職するが履歴書が薄い
        → また同じ環境
```

**数字が示す現状（公的統計・調査データ）**

| 指標 | 数値 | 出典 |
|------|------|------|
| 新卒3年以内離職率（大卒） | **34.9%** | 厚生労働省（2021年卒） |
| 20代男性の転職率 | **13.4%**（全世代最高） | マイナビ転職動向調査2025年版 |
| 20代の副業実施率 | **14.4%**（全世代最低） | パーソルキャリア2024 |
| 企業の能力開発費（GDP比） | **0.1%**（米国2.0%の20分の1） | JILPT |
| 労働生産性（時間あたり） | OECD **38か国中28位**、G7最下位 | 日本生産性本部2024 |

20代の副業実施率が全世代最低であることは重要だ。伸びしろが最大にある市場を、UNLIDが先行して開拓できる。

### 2-3. OJT崩壊——育てられない構造的理由

```
日本型雇用の前提：「終身雇用 × 年功序列」→ 先輩が長くいるから徒弟制が成立
        ↓ 前提が崩壊
① シニア社員が自分の業務で手一杯（教える余裕がない）
② 人員削減で「教える人」自体がいなくなった
③ 暗黙知伝承は、複雑化した業務には機能しない
④ OJTは「コストとして見えない」ため、削減しても気づかれにくい
        ↓
「OJTやってます」と言いながら実態は放置
＝ 若手は「成長できない」と感じて早期離職
```

### 2-4. 中小企業のAI導入遅延という機会

中小企業では「AIを使える人が社内に1〜2人しかいない → その人に全部集中してアップアップ」という状況が急増している。ランサーズ・クラウドワークスを使うという発想もない。**「AIを代わりに使って仕上げもしてくれる外部チーム」への需要は、今後急拡大する。**

## 3. 顧客定義

### 3-1. B2B：「研修を買う企業」ではなく「課題を外注したい企業」

#### ターゲット企業プロファイル

| 属性 | 詳細 |
|------|------|
| 規模 | 従業員 **30〜500名**の中小・中堅企業、スタートアップ（Sweet Spot：100〜300名） |
| 業種 | IT・コンサル・金融・サービス業（知識労働中心）。製造業は一部 |
| 経営課題 | 「人手はいないが、やらないといけない仕事が溜まっている」 |
| 意思決定者 | **経営者または事業部長**（人事部ではなく現場の責任者） |
| 予算の名目 | 業務委託費・外注費（研修費ではない） |
| 予算感 | 月3〜20万円（継続型）または案件単位50〜150万円 |

**なぜ「人事部ではなく現場の責任者」か：**
研修費は人事部予算で年度計画に縛られる。業務委託費は現場責任者が案件単位で判断できる。後者の方が意思決定が速く、成果への直結も評価しやすい。

#### 企業が感じている3つの痛み

**痛み①：「汚れ仕事・面倒な仕事を誰もやりたがらない」**
摩擦のある業務（部署間調整の下準備・クレーム後の対応設計・泥臭いリサーチ）は社内で「損な役回り」とされ、誰も手をつけない。放置すると意思決定が遅れ、関係が悪化する。

**痛み②：「外注したいが、文脈が伝わらない」**
一般的な外注先（BPO・ランサーズ）は「作業」は請け負えるが、「判断が必要な仕事」はできない。社内文脈・ステークホルダーの感情把握を伴う業務は自社でやるしかないという諦めがある。

**痛み③：「AIを活用したいが、社内にできる人が1〜2人しかいない」**
その人間が全社のAI活用の窓口になっており消耗している。「AI出力を人間らしく仕上げる」作業が大量発生しているが対応できていない。

### 3-2. B2C：「学びたい若手」ではなく「証明が欲しい若手」

#### ターゲット個人プロファイル

| 属性 | 詳細 |
|------|------|
| 年齢 | **22〜32歳** |
| 状況 | 入社1〜6年目。日々の業務はこなせるが「市場価値になっているか」の実感がない |
| 心理 | 「AIに仕事を奪われるのでは」「今の会社でしか通用しないのでは」という漠然とした不安 |
| 過去の試み | Udemy積ん読・ビジネス書翌週忘れを繰り返している |
| 本当に欲しいもの | **「自分が修羅場をくぐった」という、他者が検証可能な実績** |

#### なぜ「払う」ではなく「もらう」モデルにするか

「賃金デッドロック」に陥っている若者に「月1万円払ってください」は刺さらない。**「副業報酬をもらえて、実績も積める」が正しい答えだ。**

```
現状の若者：
  ランサーズで文字起こし → 1,000円/本 → スキルにならない
  eラーニングを買う → 積ん読 → 成長実感ゼロ
  転職活動 → 「実績が薄い」と言われる → また同じ職場

UNLIDの場合：
  クエストを完遂 → 副業報酬（ランクに応じて増加）
  + 「このフェーズで迷ったときに〇〇と判断し、FBで△△と指摘された」完遂ログ
  → 次の転職活動で「何をやってきたか」が具体的に語れる
```

**マズローの全層を同時に刺激する設計：**

| 欲求階層 | UNLIDが提供するもの |
|---------|-----------------|
| 安全欲求（どこでも生きられる） | スキルが積まれるから市場価値が上がる |
| 生理的欲求（今すぐお金が必要） | 副業報酬（ランクに応じて1,000〜5,000円/時） |
| 帰属欲求（仲間が欲しい） | コホート（3〜4人チーム）での共闘体験 |
| 承認欲求（認められたい） | 師匠から「お前はここが良かった」とFBされる |
| 自己実現欲求 | 「不確実な状況で判断して、やり遂げた」という体験 |

## 4. 市場規模

### 4-1. 3市場の交差点としてのTAM/SAM/SOM

UNLIDは単一の市場ではなく、**3つの大型市場の交差点**に位置する。

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  市場①：BPO・業務委託市場   TAM：4〜5兆円                  │
│  企業が「判断が必要な業務の解決」に払う → UNLID の受託収益  │

│                                                          │
│  市場③：人材紹介市場   TAM：3,000〜4,000億円               │
│  企業がギルドメンバーを採用 → UNLID の紹介フィー収益         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**既存プレイヤーはそれぞれ1市場しか見ていない。UNLIDは3市場を1つのエコシステムで統合する。**

## 5. 競合分析

### 5-1. 真の競合は誰か

UNLIDの真の競合は「研修会社」ではない。**BPO・クラウドソーシング・人材紹介・コンサルが真の競合だ。**

| カテゴリ | 代表企業 | UNLIDとの違い |
|---------|---------|--------------|
| **クラウドソーシング** | ランサーズ・CW | 品質保証なし。育成機能なし。「判断が必要な仕事」は取り扱えない |
| **BPO** | 大手BPO・アウトソーサー | 定型業務の標準化モデル。非定型・判断が必要な業務は不得手 |
| **零細コンサル** | 個人コンサルタント | 高品質だが高価格・個人依存。若手育成機能なし |
| **人材紹介** | リクルートエージェント | 採用「前」の実戦検証ができない |
| **研修会社** | インソース・ラーニングエージェンシー | インプットのみ。実戦完遂体験がない |
| **コーチング** | MENTO等 | 話を聞くが実力は変わらない |

### 5-2. なぜ既存プレイヤーが参入しにくいか

| 参入障壁 | 理由 |
|---------|------|
| クラウドソーシングが参入しない | 品質保証と育成は、マーケットプレイス型と構造上相容れない |
| BPOが参入しない | 非定型・判断が必要な仕事は「大量処理×標準化」モデルと相性が悪い |
| コンサルが参入しない | 「若手を育てながら解決する」は「シニアが解決する」利益構造と矛盾 |
| 人材紹介が参入しない | 採用前に実戦検証を入れると回転率が下がり短期売上が落ちる |

### 5-3. ポジショニングマップ

```
                       高品質・判断力あり
                              ↑
  大手コンサル   ●            |
                             |            ● UNLID（狙うポジション）
  零細コンサル  ●             |
                             |
個人向け ←───────────────────────────────────────── 企業向け
                             |
  コーチング    ●             |   ● 研修会社（インソース等）
  Udemy（個人）●             |   ● BPO
                             |
  ランサーズ    ●             |
                              ↓
                       低品質・作業代替
```

**UNLIDが狙うポジション：企業向け × 高品質・判断力あり × 研修会社より現場密着 × コンサルより低価格**

## 6. サービス設計——クエストとチーム制

### 6-1. クエストとは何か

クエストとは、企業が抱える「誰かがやらないといけないが手が回っていない仕事」を構造化した実務課題だ。重要なのは「正解がある作業」ではなく**「判断が必要な仕事」**であること。

クエストの評価は一貫して3軸で行われる：

```
1. 情報の網羅性：判断材料を揃える際にサボり・恐怖による放置がなかったか
2. 相手への配慮：伝え方・構成に「相手の立場への想像力」を働かせたか
3. 目的への執着：摩擦に直面したとき、逃げずに完遂したか
```

この3軸は「AIには代替されにくい人間としての誠実さ」そのものだ。

### 6-2. クエスト類型（Tier別）

#### Tier 1：即始められる（Day 1〜6ヶ月）

| クエスト | 企業の依頼例 | AI代替可能性 | 単価目安 |
|---------|-----------|------------|---------|
| **市場・競合の意思決定用分析** | 「競合が値下げした。なぜか・どう対応すべきか調べてくれ」 | 低（示唆抽出が必要） | 50〜150万円 |
| **新規取引先の信用調査＋提案設計** | 「この会社と提携したい。大丈夫か・相手が何を求めているか」 | 低（不完全情報の判断） | 30〜100万円 |
| **稟議・投資家説明用の「通る」資料** | 「事業部長の稟議が役員会で通らない。どこが問題か」 | 低（誰が読むかの想像力） | 30〜80万円 |
| **AI出力の「人間仕上げ」** | 「ChatGPTで資料構成は作れるが、役員に刺さるか分からない」 | 低（AIと補完関係） | 月3〜15万円 |
| **模擬営業ロールプレイ＋FB** | 「新入社員の営業練習の相手をしてほしい」 | ほぼ不可能 | 半日3〜8万円 |

#### Tier 2：少し深い（6ヶ月〜2年）

| クエスト | 企業の依頼例 | 単価目安 |
|---------|-----------|---------|
| **採用ブランディング＋一次調査** | 「求人を出しても応募が来ない。魅力をどう伝えればいいか」 | 50〜150万円 |
| **顧客・ユーザーインタビュー設計〜示唆抽出** | 「顧客の本音を知りたい。社員が聞くと本音が出ない」 | 80〜200万円 |
| **クレーム後の「誠実さが伝わる」対応設計** | 「取引先に迷惑をかけた。どう謝るか・再発防止をどう説明するか」 | 50〜120万円 |
| **登壇・ワークショップファシリテーション** | 「同世代の外部の人間が話した方が刺さる。登壇者を探している」 | 2〜30万円/回 |

#### Tier 3：将来の本丸（信頼が積まれてから）

| クエスト | 企業の依頼例 | 単価目安 |
|---------|-----------|---------|
| **部署間・ステークホルダー間の合意形成** | 「A部署とB部署が対立してプロジェクトが止まっている」 | 100〜300万円 |

### 6-3. チーム制（コホート）——なぜ一人でやらせないか

UNLIDの最大の設計原則：**若手を一人でクエストに送り込まない。**

```
┌──────────────────────────────────────────────────┐
│           Quest Team（3〜4名）                    │
│                                                  │
│  [調査役 / Researcher]                            │
│    情報収集・データ整理が得意                        │
│    評価：「漏れなく、逃げずに調べたか」               │
│                                                  │
│  [コミュニケーター / Connector]                    │
│    インタビュー・登壇・ファシリが得意                 │
│    評価：「本音を引き出せたか、場を作れたか」          │
│                                                  │
│  [まとめ役 / Synthesizer]                         │
│    AIを使ったドキュメント化・構成が得意               │
│    評価：「90%を100%に仕上げられたか」               │
│                                                  │
│  [問い役 / Challenger]（任意）                     │
│    「それ本当に正しいか？」と突っ込む                 │
│    評価：「チーム全体の思考の質を上げたか」            │
│                                                  │
│          ↑ 週1回：師匠からチーム単位でFB            │
└──────────────────────────────────────────────────┘
```

**チーム制が生む4つの効果：**
1. **適度なプレッシャー**：「チームに迷惑をかけられない」という自然な規律
2. **スキルの補完**：調査・コミュニケーション・まとめ・問いが役割分担される
3. **戦友感**：全員で完遂した体験が最大の継続動機
4. **相互学習**：チーム内で即座にFBし合う環境

## 7. 収益モデル

### 7-1. 全体の資金フロー

```
【企業（クライアント）】
       ↓ 業務委託費（月3〜20万円 or プロジェクト単位50〜300万円）
【UNLID】
  ├─ 若手への副業報酬（ランク制、1,000〜5,000円/時）
  ├─ 師匠のFB・監修費（羽島氏の報酬）
  └─ UNLID運営マージン（受託費の30〜40%）

【採用成立時】
  クライアント企業がギルドメンバーを採用
       ↓
  人材紹介フィー（年収×35%≒140万円/件）→ UNLID
```

### 7-2. 3つの収益源

| 収益源 | モデル | 単価感 | 役割 |
|--------|--------|--------|------|
| **B2B受託費** | 月額継続 or プロジェクト | 月3〜20万円 / 案件50〜300万円 | **メイン収益（全体の70〜80%）** |
| **人材紹介フィー** | 採用成立時成功報酬 | 年収400万×35%=**140万円/件** | **構造的差別化・高単価収益** |
| **B2B顧問契約** | 優先アクセス・月額 | 月30〜100万円 | 将来の安定収益 |

**ToCからの直接収益はゼロ。若手への支払いはB2B受託費から賄う。**

### 7-3. 価格設計と若手への報酬内訳

#### 月額モデル（エントリー）

```
エントリー（月3〜8万円）：
  若手1〜2名、月10〜20時間
  AI資料の仕上げ・調査補助・ロールプレイ等
  師匠FBは週1回のレビューのみ

  内訳（月5万円の場合）：
    若手への報酬：1万円（1名×1,000円/時×10h）
    師匠FB・監修：1.5万円
    UNLID運営マージン：2.5万円（50%）

スタンダード（月8〜20万円）：
  若手2〜3名（チーム）、月30〜60時間
  採用ブランディング・ファシリ・一次調査等
  師匠の定期FB＋中間チェック込み

  内訳（月15万円の場合）：
    若手への報酬：4.5万円（3名×1,500円/時×10h）
    師匠FB・監修：4万円
    UNLID運営マージン：6.5万円（43%）
```

#### プロジェクト型（Tier 1〜2クエスト）

```
プロジェクト型（50〜150万円/件、4〜8週間）：
  チーム4名 × 週20時間 × 6週間 = 480時間

  内訳（100万円/件の場合）：
    若手への報酬：チーム合計30〜40万円（750〜1,000円/時×480h相当）
      ※ランクに応じて個人差あり（Apprentice〜Journeyman相当）
    師匠FB・監修：20〜25万円
    UNLID運営マージン：35〜50万円（35〜50%）
```

**なぜ月額モデルが中小企業に刺さるか：**

```
× 「50万円でこれをします」（稟議が必要・一回で終わり・怖い）
○ 「月5万円で使い続けられます」（試せる・積み上がる・解約自由）
  → 月額から積み上げると年間60万円 > 単発プロジェクトより大きな収益
```

## 8. オペレーション設計——ギルドOS

### 8-1. 完遂ログの仕組み

完遂ログは単なる作業記録ではなく、「判断プロセスの記録」だ。

```
完遂ログに記録されるもの（守秘義務Tier制を適用）：

1. クエストの種別・規模感（企業名は非公開。業種・規模感・課題カテゴリのみ）
2. 自分が担当した役割・時間
3. 詰まったポイントと、どう突破したか
4. 師匠からのFB（「ここが甘い」「この判断の期待値は高かった」等）
5. 次のクエストに持ち越す改善点

→ これが積み上がると「GitHubのコントリビューション」のように
  「この人はどんな修羅場をくぐってきたか」が可視化される
```

**守秘義務Tier制（クライアント承認ベース）：**

| Tier | 公開内容 | 条件 |
|------|---------|------|
| Tier 1 | 業種・規模感・課題カテゴリのみ | 全クエストで標準適用 |
| Tier 2 | 企業名（匿名化）＋成果の概要 | クライアント承認後 |
| Tier 3 | 企業名公開＋詳細ケーススタディ | クライアントの採用ブランディング目的で使用可能 |

### 8-2. スケール設計——師匠依存からギルドOSへ

```
フェーズ1（0〜12ヶ月）：
  羽島氏が全FBを担当 → FBの「型」を言語化・記録
        ↓
フェーズ2（1〜3年）：
  「3軸評価チェックリスト」＋「AIログ解析」で評価を構造化
  実績を積んだCraftsmanをMasterに認定 → 新メンバーのFBを担当
        ↓
フェーズ3（3〜5年）：
  Master複数名体制（目標10名以上）
  羽島氏は「Masterの品質監査」に専念
  ギルドOSが自走する
```

## 9. 創業者のアンフェアアドバンテージ

### 9-1. 高成長企業での実務経験

レバレジーズ・リクルートという**国内トップクラスに密度の高い実務環境**を経験。「何が成果に直結し、何が余計な摩擦を生むか」という現場の筋を身体で学んだ。

- 研修会社創業者にありがちな「理論は語れるが現場を知らない」タイプではない
- FBが「教科書的な正解」ではなく「現場で通用する振る舞い」に基づいている
- **企業側（発注者）の言語で会話でき、Day 1から信頼を得られる**

### 9-2. 麻雀プロ——不完全情報下の意思決定のプロフェッショナル

```
麻雀の本質：
  不完全情報（相手の手牌が見えない）の中で、
  「結果論」ではなく「その瞬間の判断の期待値が正しかったか」を
  繰り返し検証する知的競技

ビジネスの摩擦業務の本質：
  不確実な状況（正解がない）の中で、
  「結果がうまくいったか」ではなく「判断プロセスが正しかったか」を
  評価する必要がある

→ この2つの構造は完全に同型
```

一般的な講師・コンサルは「正解を教える」。しかし摩擦業務に正解はない。羽島氏の指導は「正解を教える」のではなく**「判断プロセスの期待値を検証する」**。この能力は他者には容易に模倣できない。

### 9-3. 既存のネットワーク——Day 1から両面のパイプラインがある

| 面 | 状況 |
|----|------|
| **B2B側** | キャリアの繋がりから中堅企業の経営層・人事層との接点あり。「若手の定着」と「現場の汚れ仕事の放置」という痛みをヒアリング済み |
| **B2C側** | 現在進行形で若手のキャリア支援・人材紹介を行っており、この「教え子」予備軍が最初のギルドメンバー候補 |

**ゼロからの立ち上げではなく、既存の信頼関係を事業に転換するフェーズから始められる。**

## 11. リスクと対策

| リスク | 深刻度 | 内容 | 対策 |
|--------|--------|------|------|
| **労働者派遣法への抵触** | ★★★ | 実務受託の構造が「偽装請負」と判断される可能性 | **フェーズ0で弁護士・社労士による法的精査を必須実施。業務請負の要件（指揮命令・成果責任）を厳格に設計** |
| **有料職業紹介業の未取得** | ★★★ | 人材紹介フィーは職業安定法上の許可が必要 | フェーズ0〜1で申請。許可取得前は紹介フィーを収益に計上しない |
| **クエストの切り出し困難** | ★★★ | 企業が「何を外に出せるか」判断できない | 羽島氏がコンサルとして棚卸しから入る。最初の5社でクエスト類型を完成させる |
| **羽島氏への依存** | ★★★ | フェーズ1〜2はキャパがボトルネック | Master育成を最優先。2年目にMaster3名体制を目指す |
| **守秘義務・情報管理** | ★★★ | 機密情報の漏洩リスク | NDA標準化 + 守秘義務Tier制 + 情報アクセス権限設計 + 完遂ログ公開範囲の制御 |
| **若手の品質バラつき** | ★★ | チームメンバーの実力差がある | 師匠の伴走で品質担保。初期は羽島氏が直接管理。失敗もログ化 |
| **単価の妥当性証明** | ★★ | 月3〜8万円が「割高」と感じられるリスク | 最初の1〜2社は安くても実績をつくり、成果をケーススタディ化 |
| **AIの進化リスク** | ★★ | 「AI代替不可能」とした領域が3〜5年で変化する可能性 | クエスト定義を「AI代替困難度」だけでなく「企業と人間の信頼関係が必要な業務」という軸でも再定義。定期的に見直す |
| **「研修」との混同** | ★ | 市場に「研修会社」と誤解されるリスク | ブランディングで「ギルド」「実戦」の世界観を徹底。「研修」という言葉を使わない |
