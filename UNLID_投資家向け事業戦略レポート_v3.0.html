Executive Summary

UNLIDが狙うのは、日本の中小企業（SME）のAI実装を「現場に入って成果を出す」フラクショナルFDEで支援し、その監督下で副業若手ワーカーが実務を担って実績ログを積み、最終的にスカウト型人材紹介で収益化するエコシステムだ。将来はその実績ログをGitHubのビジネス職版SaaSとして外販する。

05 参入障壁——なぜ真似されにくいか

5-1. 構造的参入障壁

FDE品質の育成コスト
「現場で責任を持てるFDE」を育てるには最低1〜2年。スキルセットが再現困難
完遂ログの蓄積
1,000件以上のログがないとSaaS化の価値が出ない。先行者が圧倒的有利
両面ネットワーク効果
B2B（企業）とB2C（若手ワーカー）の両方を同時に持つ必要がある。どちらかだけでは機能しない
創業者の信頼資産
レバレジーズ出身×麻雀プロというユニークな組み合わせ。「判断プロセスのFB」に不可欠
法的設計の複雑さ
偽装請負・有料職業紹介・業務委託の3法を同時にクリアする設計が模倣コスト高

5-2. なぜ既存プレイヤーが参入しにくいか

プレイヤー参入できない構造的理由
サーキュレーション「既成プロ人材」が商品なので「若手育成」を入れると単価が下がり収益モデルが崩れる
LayerX自社ツール（Ai Workforce）がないとサービスが成立しない。SMEは対象外
顧問名鑑OBのプライドと「若手と同格」のポジションが矛盾。モデル転換不可
研修会社アウトプット（成果）での課金は事業部コストが膨大。インプット（講義）に戻る構造
クラウドソーシング品質保証と育成はマーケットプレイス型と構造上矛盾する

6-3. 完遂ログの設計

【1クエスト完了時に記録する5項目】 1. クエスト種別・業種・規模感（守秘義務Tier1） 例：「AI出力仕上げ / IT系 / 従業員50〜100名」 2. 自分の担当役割と時間 例：「まとめ役 / 約20時間」 3. 詰まったポイントと突破法（本人記述） 例：「インタビュー相手が本音を話してくれなかった。 FDEから"相手に断る理由はない"と指摘を受け 別角度で聞き直したら話してくれた」 4. FDEによる3軸スコア - 情報の網羅性：★★★★☆（4/5） - 相手への配慮：★★★☆☆（3/5） - 目的への執着：★★★★★（5/5） + 一言コメント 5. 次回改善点（本人記述） → これが積み上がると「GitHubの草（Active Contributor）」の ビジネス職版になる

守秘義務Tier制

Tier公開内容条件
Tier 1業種・規模感・課題カテゴリのみ全クエストで標準適用
Tier 2企業名（匿名化）＋成果の概要クライアント承認後
Tier 3企業名公開＋詳細ケーススタディクライアントの採用ブランディング目的

08 将来構想——完遂ログのSaaS化（GitHubのビジネス職版）

8-1. GitHubの収益モデルから学ぶ

GitHubの収益構造： 「個人に無料で使わせて習慣化させ、 その人が企業に入ったときに企業から回収する」 具体的な収益源： ① GitHub Enterprise：月21ドル/ユーザー（メイン収益） ② GitHub Copilot：月19ドル/ユーザー（急成長中） ③ GitHub Actions：従量課金 → Microsoftが2018年に約7,500億円で買収

8-2. 日本にビジネス職版GitHubは存在しない

サービス何ができるか欠点
Wantedly経験・プロジェクトを書いて採用に繋げる全部自己申告。第三者検証なし
LinkedInスキル・推薦コメントEndorsementは形骸化
YOUTRUST信頼ネットワーク×副業転職実績の「中身」は見えない
Findy/LaprasGitHub活動を集約して採用に活用エンジニア専用。ビジネス職は対象外
→「FDEが3軸スコアをつけた第三者検証付きの実績記録」はどこにも存在しない。

8-3. SaaS化の収益モデル（フェーズ4以降）

前提条件：完遂ログ累計1,000件超 ① 採用企業からの閲覧課金（GitHub Enterprise型） 「完遂ログDBを検索できる採用担当者アカウント」 → 月3〜10万円/社 ② スカウト成立時の成功報酬（フェーズ1から機能） 完遂ログ経由で採用成立 → 紹介フィー（年収×35%） ③ 企業向けSaaS（自社若手管理） 「自社の若手社員の完遂ログを社内で管理したい」 → 月5,000〜1万円/ユーザー 人事部門の研修効果可視化ニーズに対応

09 リスクと対策

9-1. 法的リスク（最重要）

リスク深刻度内容対策

競業避止義務 ★★ FDEが複数社掛け持ち時の情報持ち出し 同業種2社以上の同時担当禁止。情報アクセス権限の分離
副業ワーカーの雇用判定 ★★ 実態が雇用と判定されると労働法が適用 指揮命令の明確な排除。成果物で評価。稼働時間の管理はしない
